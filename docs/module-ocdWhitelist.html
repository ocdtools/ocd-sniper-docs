ocdWhitelist

Tracks users who signed up for the whitelist via the platform's main site popup.

Table Schema:

  • Primary Key: walletid: Wallet address of the user.

Key Features:

  • Whitelist Management: Maintains a list of users who signed up for platform access.
  • Simple Schema: Provides a lightweight table for storing basic user signup data.

Use Cases:

  • Managing access for users signing up through the whitelist.
  • Tracking wallet addresses for exclusive campaigns or feature rollouts.

This module represents the ocdWhitelist table schema and provides information about users on the whitelist.

Table Definitions

ocdWhitelist

Description:

  • The ocdWhitelist table stores a list of users who signed up to be on the platform's waiting list. This list is generated from a sign-up popup on the main site and serves as a preliminary collection of interested users for potential onboarding or beta access.

Schema:
Name Type Description
walletid string

Wallet address of the user who signed up for the whitelist (Primary Key).
