ocdBetaUsers

Manages beta users and their access levels, including onboarding statuses and referral-based prioritization.

Table Schema:

  • Primary Key: userid: Unique identifier for the user, wallet address.
  • Attributes:
    • referredby: Referral code used during signup, if applicable.
    • betatype: Type of user (admin, KOL, whitelist).
    • accesslevel: Access level (full, restricted, trial).
    • timestamp: Time the beta user was added.

Key Features:

  • Dynamic Access Control: Manages user permissions and access levels.
  • Referral Integration: Links beta users with referral codes for tracking.
  • Program Monitoring: Tracks enrollment and onboarding processes.

Use Cases:

  • Managing beta access for platform features.
  • Prioritizing KOLs or referred users for early access.
  • Tracking onboarding progress for beta users.

This module represents the ocdBetaUsers table schema and provides data for managing beta program enrollment.

Table Definitions

ocdBetaUsers

Description:

  • The ocdBetaUsers table serves as a whitelist of users who are allowed to access the platform. Users are added to this table after signing up, connecting, verifying, and authenticating their wallet. During this process, referral codes are validated, and several downstream updates are triggered via the Lambda function.
