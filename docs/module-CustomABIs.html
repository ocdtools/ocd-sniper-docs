CustomABIs

Exports ABIs for custom protocols, including ERC-20 standard tokens and the OCD router.

Included ABIs:

  • erc20ABI: ABI for interacting with ERC-20 token contracts.
  • ocdRouterABI: ABI for interacting with the OCD router for custom token routing.

Key Features:

  • ERC-20 Compatibility: Supports token transfers, balances, and allowances.
  • Custom Routing: Provides advanced routing capabilities for tokens.

Members

(inner, constant) erc20ABI :Array.<Object>

Description:

  • ABI for interacting with standard ERC-20 token contracts, including methods for transferring tokens, checking balances, and managing allowances.

    Functions:

    • View Functions:

      • name(): Returns the name of the token (e.g., "MyToken").
      • symbol(): Returns the symbol of the token (e.g., "MTK").
      • decimals(): Returns the number of decimals the token uses (e.g., 18).
      • totalSupply(): Returns the total supply of the token.
      • balanceOf(address _owner): Returns the token balance of a specific address.
      • allowance(address _owner, address _spender): Returns the remaining tokens that _spender is allowed to spend on behalf of _owner.

    • Write Functions:

      • approve(address _spender, uint256 _value): Approves _spender to transfer up to _value tokens on behalf of the caller.
      • transfer(address _to, uint256 _value): Transfers _value tokens to the _to address.
      • transferFrom(address _from, address _to, uint256 _value): Transfers _value tokens from _from to _to using an allowance.

    Events:

    • Approval:
      • Emitted when the approve function is called.
      • Inputs:
        • owner (address): The address of the token owner.
        • spender (address): The address approved to spend the tokens.
        • value (uint256): The amount approved.
    • Transfer:
      • Emitted when a transfer occurs.
      • Inputs:
        • from (address): The address sending the tokens.
        • to (address): The address receiving the tokens.
        • value (uint256): The amount transferred.

    Key Features:

    • ERC-20 Standard: Implements the widely used ERC-20 standard for fungible tokens.
    • Allowance System: Supports an approval mechanism for third-party token spending.
    • Transfer Events: Logs Approval and Transfer events for transaction transparency.
See:
  • EIP-20: ERC-20 Token Standard

(inner, constant) ocdRouterABI :Array.<Object>

Description:

  • ABI for interacting with the OCD router contracts, supporting custom token routing, swaps, and utility functions.

    Functions:

    • Constructor:

      • Initializes the contract with _router (address): The initial router address.

    • View Functions:

      • factory(): Returns the address of the factory contract used by the router.
      • isBlocked(address): Checks if a specific address is blocked.

    • Write Functions:

      • getProfit(address dexRouter, address tokenIn, uint256 amountIn, address tokenOut, uint256 minAmountOut):
        • Calculates profit from a token swap using the specified DEX router.
      • setFee(uint256 _buyFee, uint256 _sellFee):
        • Updates the buy and sell fees.
      • setFeeReceiver(address _recvr):
        • Sets the address that will receive transaction fees.
      • setRouterStatus(address _router, bool _status):
        • Enables or disables a router for swaps.
      • snipeToken(address dexRouter, address tokenIn, address tokenOut, uint256 minAmountOut):
        • Executes a token snipe operation, allowing for quick trades based on a minimum output.
      • transferOnwer(address _ownerAddr):
        • Transfers ownership of the contract to a new address.
      • withdrawStuckETH(uint256 amount):
        • Withdraws stuck Ether from the contract.
      • withdrawStuckTokens(uint256 amount, address token):
        • Withdraws stuck tokens from the contract.

    Key Features:

    • Custom Routing: Enables flexible token routing and swapping.
    • Fee Management: Supports configuration of transaction fees and fee recipients.
    • Sniping Functionality: Includes specialized functionality for executing sniping trades.
    • Rescue Operations: Allows retrieval of stuck ETH and tokens.
