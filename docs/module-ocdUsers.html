ocdUsers

Stores user-specific preferences and configurations for platform features, including sniper tools and watchlists.

Table Schema:

  • Primary Key: userid: Unique identifier for the user, wallet address.
  • Attributes:
    • referredby: Referral code used during signup.
    • ethsettings: Ethereum sniper settings configured by the user.
    • solsettings: Solana sniper settings configured by the user.
    • tronsettings: Tron sniper settings configured by the user.
    • amounts: Financial preferences or limits defined by the user.
    • timestamp: Time the user was added to the platform.

Key Features:

  • Personalized Configuration: Allows users to customize sniper tools for different chains.
  • Referral Tracking: Links users with their referrer for analytics and rewards.
  • Real-Time Updates: Synchronizes user settings dynamically across the platform.

Use Cases:

  • Customizing sniper tool settings for Ethereum, Solana, and Tron.
  • Tracking user preferences for watchlists and alerts.
  • Integrating user-specific configurations into platform features.

This module represents the ocdUsers table schema and provides detailed information about user settings.

Table Definitions

ocdUsers

Description:

  • The ocdUsers table stores a list of authenticated users and their associated settings for blockchain sniping. Users are added to this table after successful authentication, along with their referral information and specific sniper configurations for Ethereum, Solana, and Tron.
