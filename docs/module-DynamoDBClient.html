DynamoDBClient

Provides utility functions to manage and interact with DynamoDB clients.

  • Client Initialization:

    • getDynamoDBRawClient: Initializes and retrieves the raw DynamoDB client for low-level operations.
    • getDynamoDBDocClient: Initializes and retrieves the DynamoDB Document client for high-level operations.

  • Retry Logic:

    • Implements retry logic with exponential backoff for client initialization using withRetry.

  • Resource Management:

    • closeAllDynamoDBClients: Gracefully closes all active DynamoDB client instances.

  • Internal Utilities:

    • setupClientListeners: Adds listeners to monitor client initialization status.
    • getDynamoDBInstance: Handles lazy initialization of raw or document clients.

This module ensures efficient and reusable DynamoDB client management with features like retry logic, resource cleanup, and JSON data marshalling.

Methods

(async, static) closeAllDynamoDBClients() → {Promise.<void>}

Description:

  • Gracefully closes all active DynamoDB client instances.

    • Iterates through all cached DynamoDB client instances.
    • Calls destroy on each client to release resources.
    • Logs information about successful closures and errors for failures.
Returns:

Resolves when all clients have been closed successfully.

(async, static) getDynamoDBDocClient() → {Promise.<DynamoDBDocumentClient>}

Description:

  • Returns the DynamoDB Document client for high-level operations.

    • Uses getDynamoDBInstance to lazily initialize the document client if not already created.
    • Designed for easier interaction with DynamoDB, including JSON data marshalling/unmarshalling.
Throws:

Throws if the document client fails to initialize.

Returns:

Resolves with the DynamoDB Document client instance.

(async, inner) getDynamoDBInstance(clientType) → {Promise.<(DynamoDBClient|DynamoDBDocumentClient)>}

Description:

  • Lazily initializes and returns a DynamoDB client or document client.

    • Creates a new client if not already initialized.
    • Ensures singleton behavior for each client type.
    • Supports both raw and document clients, depending on the requested type.

(async, inner) withRetry(initFn, retriesopt) → {Promise.<any>}

Description:

  • Implements exponential backoff retry logic for DynamoDB client initialization.

    • Attempts to initialize the client up to a maximum number of retries.
    • Uses exponential backoff to delay subsequent retries, capped at a maximum delay.
    • Logs warnings for each retry attempt and errors if all retries fail.
