ocdWallets

Organizes and secures user wallets with multi-chain support, including visibility settings and encryption.

Table Schema:

  • Primary Key: userid: Unique identifier for the user, wallet address.
  • Sort Key: walletid: Wallet address associated with the user.
  • Attributes:
    • name: Name assigned to the wallet.
    • chain: Blockchain the wallet is associated with.
    • key: Encrypted private key for the wallet.
    • hiddenChains: List of blockchains hidden or excluded by the user.

Key Features:

  • Multi-Chain Support: Manages wallets across multiple blockchains.
  • Encryption: Securely stores private keys for user wallets.
  • Customization: Allows users to manage visibility settings for wallets.

Use Cases:

  • Managing user wallets across Ethereum, Solana, and other chains.
  • Securing private keys with robust encryption.
  • Customizing wallet visibility and usage preferences.

This module represents the ocdWallets table schema and provides detailed information about wallet management.

Table Definitions

ocdWallets

Description:

  • The ocdWallets table maintains a list of wallets associated with each user on the platform. Each user can have multiple wallets across different blockchains. This table stores wallet-specific details, such as chain type, private keys, and user preferences.
