ocdWatchlist

Tracks tokens and liquidity pools favorited by users for real-time monitoring and notifications.

Table Schema:

  • Primary Key: userid: Unique identifier for the user, wallet address.
  • Sort Key: tokenid: Address of the token added to the watchlist.
  • Attributes:
    • token: Details about the favorited token.
    • image: URL of the token image for display purposes.
    • details: Metadata about the token's pool or trading pair.
    • chain: Blockchain the token is associated with.
    • timestamp: Time the token was added to the watchlist.

Key Features:

  • Token Tracking: Enables users to monitor their favorite tokens and pools.
  • Real-Time Notifications: Alerts users to changes in favorited tokens or pools.
  • Multi-Chain Support: Tracks tokens across multiple blockchains.

Use Cases:

  • Monitoring liquidity pools or tokens of interest.
  • Providing users with notifications and alerts for their watchlist items.
  • Supporting multi-chain token tracking for enhanced user experience.

This module represents the ocdWatchlist table schema and provides detailed information about user watchlists.

Table Definitions

ocdWatchlist

Description:

  • The ocdWatchlist table tracks a list of tokens favorited by users on the platform. Each entry represents a specific token saved by a user, along with its details and metadata, enabling users to quickly access and monitor their favorite tokens.
