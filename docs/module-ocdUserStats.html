ocdUserStats

Aggregates and manages user-specific statistics, including trade volumes, counts, referral metrics, and engagement details.

Table Schema:

  • Primary Key: userid: Unique identifier for the user, wallet address.
  • Attributes:
    • totalpoints: Total leaderboard points accumulated by the user.
    • tradevolume: Total trade volume by the user.
    • totalvalue: Total value of trades in ETH.
    • totalvalueusd: Total value of trades in USD.
    • totaltradecount: Total number of trades.
    • lasttradetime: Timestamp of the last trade made by the user.
    • referralcount: Number of referrals associated with the user.

Key Features:

  • Trade Metrics: Tracks user-specific trade volumes, values, and counts.
  • Referral Metrics: Records referral activity for each user.
  • Leaderboard Points: Calculates points for leaderboard ranking.
  • Real-Time Updates: Reflects live user activity using DynamoDB Streams.

Use Cases:

  • Generating user-specific dashboards or reports.
  • Analyzing individual user engagement and activity.
  • Supporting leaderboard ranking and incentive programs.

Supported Data Types:

  • Trade Metrics: Volume, value (ETH/USD), and counts.
  • Referral Metrics: Number of referred users.

This module represents the ocdUserStats table schema and provides information about individual user statistics on the platform.

Table Definitions

ocdUserStats

Description:

  • The ocdUserStats table stores aggregated totals and statistics for individual users based on their activity on the platform. It is designed to support analytics, user performance tracking, and leaderboard rankings.
