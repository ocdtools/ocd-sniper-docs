EthereumTransactions

Provides comprehensive utilities for processing, decoding, constructing, and monitoring Ethereum transactions.

This module includes the following utilities:

  • constructTxDetails: Constructs transaction details from function names and decoded parameters.
  • decodeTransaction: Decodes transaction data based on ABI-defined input details.
  • determineTokenAddress: Identifies the token address involved in UniswapV2 transactions.
  • extractTransactionData: Extracts and decodes UniswapV2 transaction data.
  • getBuyReceipts: Processes transaction receipts asynchronously, supporting test mode with mock data.
  • waitForTransactionReceipt: Waits for a transaction receipt with a configurable timeout, polling at 1-second intervals.
  • waitForTargetBlock: Monitors the Ethereum blockchain until a specified block number is mined.

These utilities are essential for handling Ethereum transactions, including UniswapV2 interactions, transaction decoding, receipt processing, and block confirmations.

Description:

  • Constructs transaction details based on the function name and decoded parameters.

    This function processes the details of Ethereum transactions, extracting relevant information based on the function being executed. It supports transactions involving liquidity addition (with ETH or tokens) and can be extended for other function types.

The name of the function being executed in the transaction.

  • quoteToken: The address of the quote token.
  • tokenAddress: The address of the token involved in the transaction.
methodId Object

The method ID of the transaction.
tx Object

The original transaction object from the blockchain.
Returns:

An object containing:

  • error {boolean} - Indicates whether an error occurred during construction.
  • txDetails {Object|null} - The constructed transaction details (null if an error occurred).
  • message {string|null} - An error message, if applicable.

(static) decodeTransaction(methodId, txData, tx) → {Object}

Description:

  • Decodes transaction data using precomputed input parameter details.

    This function determines the corresponding function for a given method ID, decodes the transaction data using the ABI specifications, and constructs a detailed representation of the transaction. It performs the following steps:

    1. Identifies the function associated with the provided method ID.
    2. Decodes the transaction parameters using ABI-defined input details.
    3. Constructs additional transaction details for further processing.
Parameters:
Name Type Description
methodId string

The method ID of the transaction (first 4 bytes of the calldata).
txData string

The raw transaction data (excluding the method ID).
tx Object

The original transaction object, containing metadata like hash.
Returns:

An object containing:

  • error {boolean} - Indicates whether the decoding was successful.
  • txDetails {Object|null} - Decoded transaction details (null if an error occurred).
  • message {string} - Error or success message describing the outcome.
Type
Object

(static) determineTokenAddress(functionName, decodedParams, quoteTokenopt) → {Object}

Description:

  • Determines the token address based on the function name and decoded parameters.

    This function identifies the relevant token address from the transaction's decoded parameters, specifically for UniswapV2 functions such as addLiquidityETH or addLiquidity. It processes the parameters to extract the non-quote token address or validates the provided token address in the transaction data.

Parameters:
Name Type Attributes Default Description
functionName string

The name of the UniswapV2 function being processed (e.g., addLiquidityETH or addLiquidity).
decodedParams Object

Decoded transaction parameters, including token addresses.
quoteToken string <optional>
 WETH

The address of the quote token (default is WETH).
Returns:

An object containing:

  • error {boolean} - Indicates whether determining the token address was successful.
  • tokenAddress {string|null} - The identified token address (null if an error occurred).
  • message {string} - Error or success message describing the outcome.
  • quoteToken {string|null} - The quote token address, if relevant.

Description:

  • Extracts and decodes transaction data for UniswapV2 transactions.

    This function validates whether a transaction targets the UniswapV2 router, extracts the method ID and transaction data, decodes the transaction details, and returns the relevant information or an error message if the decoding fails.

  • txDetails {Object|null} - The decoded transaction details (null if an error occurred).
  • message {string} - Error or success message describing the result.

(async, generator, static) getBuyReceipts(transactions, testModeopt) → {Promise.<Object>}

Description:

  • Processes transaction receipts for buy transactions.

    • In test mode, mock receipts are returned for testing purposes.
    • In normal mode, it waits for transaction receipts and yields results.
    • Useful for asynchronous processing of multiple transactions.
Parameters:
Name Type Attributes Default Description
transactions Array.<Object>

Array of submitted transactions, each containing:

  • tx {Object}: The transaction object (with wait and hash properties).
  • walletName {string}: Name of the wallet associated with the transaction.
  • walletAddress {string}: Address of the wallet associated with the transaction.
testMode boolean <optional>
 false

If true, mock receipts are returned for testing purposes.
Yields:
Yields an object for each transaction containing: - `error` {boolean}: Indicates if an error occurred during receipt processing. - `receipt` {Object|null}: The transaction receipt (or `null` if an error occurred). - `walletName` {string}: Name of the wallet associated with the transaction. - `walletAddress` {string}: Address of the wallet associated with the transaction. - `txHash` {string}: Transaction hash (if available). - `message` {string}: A descriptive message for the processing result. - `context` {Object|null}: Additional error details (if applicable).
Type
Promise.<Object>

(async, static) waitForTargetBlock(targetBlock, pollIntervalopt) → {Promise.<Object>}

Description:

  • Waits for the target block to be mined on the Ethereum blockchain.

    • Polls the current block number at a specified interval until the target block is reached.
    • Uses the Ethereum RPC provider to fetch the current block number.
    • Returns a success message when the target block is reached or an error message if something goes wrong.
Parameters:
Name Type Attributes Default Description
targetBlock number

The block number to wait for.
pollInterval number <optional>
 12000

Polling interval in milliseconds (default: 12 seconds).
Throws:

Logs and propagates unexpected errors encountered during the process.

Type
Error
Returns:

Resolves to an object containing:

  • error {boolean}: Indicates whether an error occurred.
  • message {string}: Success or error message describing the result.
Type
Promise.<Object>

(async, static) waitForTransactionReceipt(txHash, timeoutopt) → {Promise.<Object>}

Description:

  • Waits for the transaction receipt within a specified timeout.

    • Polls for the transaction receipt at 1-second intervals.
    • Resolves as soon as the receipt is found or the timeout is reached.
    • Logs debug messages for progress and errors.
Parameters:
Name Type Attributes Default Description
txHash string

The transaction hash to fetch the receipt for.
timeout number <optional>

Timeout duration in milliseconds (default: 60 seconds).
Returns:

  • receipt {Object|null}: The transaction receipt (or null if an error occurred or timeout).
  • message {string}: A descriptive message for the result.
  • context {Object|null}: Additional error details (if applicable).
