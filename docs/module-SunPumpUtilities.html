SunPumpUtilities

Provides utilities and handlers for processing events, configurations, and token details related to the SunPump contract on the Tron network.

This module includes the following utilities:

  • filterPendingConfigs: Filters in-memory pending configurations based on chain, token address, and method ID.
  • handleSunPumpConfig: Handles SunPump configuration for liquidity pool transactions, including slippage calculations.
  • handleTokenCreateEvent: Processes "TokenCreate" events, retrieving and validating liquidity, token information, and configurations.
  • processSunPumpConfigs: Processes configurations for SunPump liquidity pool transactions and marks them inactive upon completion.
  • processTronEvents: Fetches and processes SunPump contract events for specific block ranges, handling instructions for token creation or liquidity updates.

These utilities are designed for use cases involving SunPump liquidity analysis, token event processing, and Tron DEX interactions.

Methods

(async, static) exports.processTronEvents(currentBlockNumber) → {Promise.<number>}

Description:

  • Processes events emitted by the SunPump contract for a specific block range.

    • Fetches events from the SunPump contract for the provided block number.
    • Processes each event, logging details and handling active instructions.
    • Returns the total count of events successfully processed.
Parameters:
Name Type Description
currentBlockNumber number

The current block number to fetch and process events for.
Throws:

Logs and handles any errors during event processing. Rate limit errors are logged but do not throw.

Type
Error
Returns:

The total number of events processed.

Type
Promise.<number>

(static) filterPendingConfigs(pendingConfigs, chain, tokenAddress, methodId) → {Object}

Description:

  • Filters pending configurations based on the given chain, token address, and method ID.

    • Validates inputs to ensure correct types and structure.
    • Retrieves configurations for the specified token and chain.
    • Filters configurations by the specified method ID.
Parameters:
Name Type Description
pendingConfigs Object

An in-memory object containing configurations.
chain string

The blockchain name (e.g., "ethereum", "tron").
tokenAddress string

The token address to filter configurations for.
methodId string

The method ID to filter configurations by.
Throws:

Returns an error object if unexpected issues arise.

Type
Error
Returns:

An object containing the following properties:
