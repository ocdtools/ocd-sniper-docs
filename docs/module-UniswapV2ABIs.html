UniswapV2ABIs

Exports ABIs for interacting with Uniswap V2 contracts, including routers, pairs, and factories.

Included ABIs:

  • uniswapV2PairABI: ABI for Uniswap V2 pair contracts (liquidity pools).
  • uniswapV2RouterABI: ABI for Uniswap V2 router contracts (token swaps and routing).
  • uniswapV2FactoryABI: ABI for Uniswap V2 factory contracts (pair creation).

Key Features:

  • DeFi Operations: Provides tools for token swaps and liquidity management.
  • Ecosystem Growth: Supports new pair creation and liquidity incentives.

Members

(inner, constant) uniswapV2FactoryABI :Array.<Object>

Description:

  • ABI for interacting with Uniswap V2 factory contracts, enabling the creation and management of liquidity pairs.

    Constructor:

    • Inputs:
      • _feeToSetter (address): Address responsible for setting the fee collection address.

    Events:

    • PairCreated:
      • Triggered when a new liquidity pair is created.
      • Inputs:
        • token0 (address): Address of the first token in the pair.
        • token1 (address): Address of the second token in the pair.
        • pair (address): Address of the created pair contract.
        • uint256: Index of the created pair.

    Functions:

    • Pair Management:

      • createPair(tokenA, tokenB):
        • Creates a new liquidity pair for the given tokens.
        • Outputs: pair (address): Address of the newly created pair.
      • getPair(tokenA, tokenB):
        • Returns the address of the pair for the specified tokens if it exists.
        • Outputs: pair (address).

    • Viewing All Pairs:

      • allPairs(index):
        • Returns the address of the pair at the specified index.
        • Outputs: pair (address).
      • allPairsLength():
        • Returns the total number of pairs created by the factory.
        • Outputs: length (uint256).

    • Fee Management:

      • feeTo():
        • Returns the current address where fees are sent.
        • Outputs: feeTo (address).
      • feeToSetter():
        • Returns the current address responsible for setting the fee collection address.
        • Outputs: feeToSetter (address).
      • setFeeTo(_feeTo):
        • Sets the address where fees are sent.
        • Inputs:
          • _feeTo (address): New fee collection address.
      • setFeeToSetter(_feeToSetter):
        • Updates the address responsible for setting the fee collection address.
        • Inputs:
          • _feeToSetter (address): New feeToSetter address.

    Key Features:

    • Liquidity Pair Creation: Facilitates the creation of new token pairs for Uniswap V2.
    • Fee Management: Allows configuration of fee collection and associated administrative roles.
    • Pair Retrieval: Provides methods to query existing pairs by token addresses or index.

(inner, constant) uniswapV2PairABI :Array.<Object>

Description:

  • ABI for interacting with Uniswap V2 pair contracts, enabling operations on liquidity pools like token swaps, liquidity provision, and synchronization.

    Events:

    • Approval:
      • Emitted when an approval is set by the owner to a spender.
      • Inputs:
        • owner (address): Address of the owner.
        • spender (address): Address of the spender.
        • value (uint256): Amount approved.
    • Burn:
      • Emitted when liquidity is burned, and tokens are sent back to the user.
      • Inputs:
        • sender (address): Address initiating the burn.
        • amount0 (uint256): Amount of token0 burned.
        • amount1 (uint256): Amount of token1 burned.
        • to (address): Address receiving the tokens.
    • Mint:
      • Emitted when liquidity is added to the pool.
      • Inputs:
        • sender (address): Address initiating the mint.
        • amount0 (uint256): Amount of token0 added.
        • amount1 (uint256): Amount of token1 added.
    • Swap:
      • Emitted during a swap operation.
      • Inputs:
        • sender (address): Address initiating the swap.
        • amount0In (uint256): Amount of token0 input.
        • amount1In (uint256): Amount of token1 input.
        • amount0Out (uint256): Amount of token0 output.
        • amount1Out (uint256): Amount of token1 output.
        • to (address): Recipient of the swap.
    • Sync:
      • Emitted when the reserves of the pair are updated.
      • Inputs:
        • reserve0 (uint112): Reserve of token0.
        • reserve1 (uint112): Reserve of token1.
    • Transfer:
      • Emitted when tokens are transferred.
      • Inputs:
        • from (address): Address sending the tokens.
        • to (address): Address receiving the tokens.
        • value (uint256): Amount of tokens transferred.

    Functions:

    • View Functions:
      • DOMAIN_SEPARATOR(): Returns the domain separator for EIP-712 compatibility.
      • MINIMUM_LIQUIDITY(): Returns the minimum liquidity for the pair.
      • getReserves(): Returns the current reserves of token0 and token1.
      • token0(): Returns the address of token0.
      • token1(): Returns the address of token1.
      • totalSupply(): Returns the total supply of liquidity tokens.
    • Write Functions:
      • mint(address to): Mints liquidity tokens and sends them to the specified address.
      • burn(address to): Burns liquidity tokens and sends the underlying assets to the specified address.
      • swap(uint256 amount0Out, uint256 amount1Out, address to, bytes data): Executes a swap operation.
      • sync(): Synchronizes the reserves of the pair.
    • Utility Functions:
      • initialize(address _token0, address _token1): Initializes the pair with the specified tokens.
      • permit(address owner, address spender, uint256 value, uint256 deadline, uint8 v, bytes32 r, bytes32 s): Allows an address to approve a spender via signature.

    Key Features:

    • Liquidity Management: Supports minting and burning of liquidity tokens.
    • Swaps: Handles token swaps with detailed inputs and outputs.
    • Reserves Synchronization: Ensures pair reserves are up to date.
    • Approval and Transfer: Implements ERC-20 token standards for approval and transfer.

(inner, constant) uniswapV2RouterABI :Array.<Object>

Description:

  • ABI for interacting with Uniswap V2 router contracts, supporting token swaps, liquidity management, and other DeFi operations.

    Constructor:

    • Inputs:
      • _factory (address): Address of the Uniswap factory.
      • _WETH (address): Address of the wrapped Ether (WETH) token.

    Functions:

    • Liquidity Management:

      • addLiquidity(tokenA, tokenB, amountADesired, amountBDesired, amountAMin, amountBMin, to, deadline):
        • Adds liquidity to a pool.
        • Outputs: amountA (uint256), amountB (uint256), liquidity (uint256).
      • addLiquidityETH(token, amountTokenDesired, amountTokenMin, amountETHMin, to, deadline):
        • Adds liquidity to a pool involving ETH.
        • Outputs: amountToken (uint256), amountETH (uint256), liquidity (uint256).
      • removeLiquidity(tokenA, tokenB, liquidity, amountAMin, amountBMin, to, deadline):
        • Removes liquidity from a pool.
        • Outputs: amountA (uint256), amountB (uint256).
      • removeLiquidityETH(token, liquidity, amountTokenMin, amountETHMin, to, deadline):
        • Removes liquidity involving ETH.
        • Outputs: amountToken (uint256), amountETH (uint256).

    • Token Swaps:

      • swapExactTokensForTokens(amountIn, amountOutMin, path, to, deadline):
        • Swaps an exact amount of input tokens for output tokens.
        • Outputs: amounts (uint256[]).
      • swapTokensForExactTokens(amountOut, amountInMax, path, to, deadline):
        • Swaps tokens to receive an exact amount of output tokens.
        • Outputs: amounts (uint256[]).
      • swapExactETHForTokens(amountOutMin, path, to, deadline):
        • Swaps ETH for an exact amount of tokens.
        • Outputs: amounts (uint256[]).
      • swapExactTokensForETH(amountIn, amountOutMin, path, to, deadline):
        • Swaps tokens for an exact amount of ETH.
        • Outputs: amounts (uint256[]).
      • swapTokensForExactETH(amountOut, amountInMax, path, to, deadline):
        • Swaps tokens to receive an exact amount of ETH.
        • Outputs: amounts (uint256[]).

    • View Functions:

      • WETH(): Returns the WETH address.
      • factory(): Returns the address of the Uniswap factory.
      • getAmountsIn(amountOut, path): Calculates the input token amounts required for a desired output.
        • Outputs: amounts (uint256[]).
      • getAmountsOut(amountIn, path): Calculates the output token amounts for a given input.
        • Outputs: amounts (uint256[]).
      • quote(amountA, reserveA, reserveB): Returns a quote for a token amount based on reserves.
        • Outputs: amountB (uint256).

    • Other Operations:

      • receive(): Fallback function to accept ETH transfers.
      • getAmountIn(amountOut, reserveIn, reserveOut):
        • Calculates the amount of input tokens required for a given output.
        • Outputs: amountIn (uint256).
      • getAmountOut(amountIn, reserveIn, reserveOut):
        • Calculates the amount of output tokens for a given input.
        • Outputs: amountOut (uint256).

    Key Features:

    • Liquidity Provision and Removal: Handles adding and removing liquidity with and without ETH.
    • Flexible Token Swaps: Supports swaps with slippage protection and fee-on-transfer tokens.
    • Price Calculation: Provides functions for quoting prices and calculating token amounts.
