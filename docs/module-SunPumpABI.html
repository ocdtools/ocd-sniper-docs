SunPumpABI

Exports the ABI for the Sun Pump contract, used for specific DeFi mechanics and custom operations.

Included ABIs:

  • sunPumpContractABI: ABI for the Sun Pump contract.

Key Features:

  • Custom DeFi Operations: Facilitates specialized contract interactions for pumping mechanisms.

Members

(inner, constant) sunPumpContractABI :Array.<Object>

Description:

  • ABI for interacting with the Sun Pump contract, enabling operations related to specific DeFi mechanics such as token launches, fee management, and token trading.

    Events:

    • LaunchPending:
      • Emitted when a token is pending launch.
      • Inputs:
        • token (address): The address of the pending token.
    • LauncherChanged:
      • Emitted when the launcher address changes.
      • Inputs:
        • oldLauncher (address): The previous launcher address.
        • newLauncher (address): The new launcher address.
    • MinTxFeeSet and MintFeeSet:
      • Emitted when minimum transaction or mint fees are updated.
      • Inputs:
        • oldFee (uint256): The previous fee.
        • newFee (uint256): The updated fee.
    • OperatorChanged:
      • Emitted when the operator address changes.
      • Inputs:
        • oldOperator (address): The previous operator.
        • newOperator (address): The new operator.
    • OwnerChanged and PendingOwnerSet:
      • Emitted when the contract ownership changes or a new owner is set as pending.
    • PurchaseFeeSet and SaleFeeSet:
      • Emitted when purchase or sale fees are updated.
    • TokenCreate and TokenLaunched:
      • Emitted during token creation and launch.
      • Inputs:
        • tokenAddress (address): Address of the token.
        • tokenIndex (uint256): The index of the token in the system.
        • creator (address): Address of the token creator.

    Functions:

    • View Functions:
      • LAUNCH_FEE(): Returns the launch fee.
      • TOKEN_SUPPLY(): Returns the total token supply.
      • getPrice(address token): Returns the current price of the specified token.
      • getTrxAmountBySale(address token, uint256 tokenAmount): Returns the amount of TRX for a given token sale amount.
    • Write Functions:
      • initialize(address _vault, address _v2Router, uint256 _salefee, uint256 _purchasefee): Initializes the contract with the specified parameters.
      • purchaseToken(address token, uint256 AmountMin): Allows a user to purchase tokens with a minimum amount of TRX.
      • saleToken(address token, uint256 tokenAmount, uint256 AmountMin): Allows a user to sell tokens with a minimum TRX amount received.
      • setMinTxFee(uint256 newFee): Updates the minimum transaction fee.
      • setMintFee(uint256 newFee): Updates the minting fee.

    Key Features:

    • Token Launching: Supports launching new tokens with configurable parameters.
    • Dynamic Fee Management: Allows setting and updating various fee types.
    • Trading Operations: Includes functions for purchasing and selling tokens, with detailed fee structures.
    • Event Logging: Comprehensive event tracking for key operations.
